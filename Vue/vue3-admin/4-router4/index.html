前言

vue-router是vue全家桶必备工具，vue3.x使用 vue-router 必须使用 4+版本才可以了。具体可以查看的 vue-router@4+文档 。

安装vue-router@4

首先安装依赖包，并指定大版本为 4+

# yarn
yarn add vue-router@next

# npm
npm install vue-router@next

页面准备

我们在页面中创建几个基础的页面，@/pages/dashboard/analysis.vue(仪表盘页面) 、@/pages/system/account.vue（账户页面）、@/pages/about/index.vue（关于页面）、@/pages/exception/index.vue（错误页，404，500等）。以错误页面为例子，简单写了些逻辑：

<template>
  <div>
    {{currentStatus}}
  </div>
</template>

<script lang="ts">
import { computed, defineComponent, PropType, ref, unref } from 'vue'
import { useRoute } from 'vue-router'

enum ExceptionEnum {
  PAGE_NOT_ACCESS = 403,
  PAGE_NOT_FOUND = 404,
  NET_WORK_ERROR = 500,
}

interface MapValue {
  title: string;
  subTitle: string;
}

export default defineComponent({
  props:{
    // 状态码
    status: {
      type: Number as PropType<number>,
      default: ExceptionEnum.PAGE_NOT_FOUND,
    },
    title: {
      type: String as PropType<string>,
      default: '',
    },
    subTitle: {
      type: String as PropType<string>,
      default: '',
    },
  },
  setup (props) {
    const { query } = useRoute();
    const statusMapRef = ref(new Map<string | number, MapValue>());

    statusMapRef.value.set(ExceptionEnum.PAGE_NOT_ACCESS, {
      title: '403',
      subTitle: '无权限操作',
    })

    statusMapRef.value.set(ExceptionEnum.PAGE_NOT_FOUND, {
      title: '404',
      subTitle: '找不到该页面',
    })

    statusMapRef.value.set(ExceptionEnum.NET_WORK_ERROR, {
      title: '500',
      subTitle: '服务器异常',
    })

    const currentStatus = computed<MapValue>(() => {
      const { status: routeQueryStatus } = query;
      const { status } = props;
      const code = Number(routeQueryStatus) || status;
      return statusMapRef.value.get(code) as MapValue;
    })

    return {
      currentStatus
    }
  }
})
</script>

这里能看到，vue-router@4 提供了一些钩子，用来访问路由useRouter()和当前路由useRoute()等。

注册路由

在 src/router 下新建 index.ts 文件，内容如下：

import type { App } from 'vue';
import type { RouteRecordRaw } from 'vue-router';
import { createRouter, createWebHashHistory } from 'vue-router';

const LayoutRoot = () => import('@/layouts/root/index.vue')

// 页面name枚举
export enum PageRouterName {
  Root = 'Root',
  Login = 'Login',
  Dashboard = 'Dashboard',
  DashboardAnalysis = 'DashboardAnalysis',
  System = 'System',
  SystemAccount = 'SystemAccount',
  About = 'About',
  ErrorPage = 'ErrorPage'
}

const routes: RouteRecordRaw[] = [
  {
    path: '/',
    name: PageRouterName.Root,
    redirect: '/dashboard',
    meta: {
      title: '首页'
    },
  },
  {
    path: '/login',
    name: PageRouterName.Login,
    component: () => import('@/pages/login/index.vue'),
    meta: {
      title: '登录'
    }
  },
  {
    path: '/dashboard',
    name: PageRouterName.Dashboard,
    component: LayoutRoot,
    redirect: '/dashboard/analysis',
    meta: {
      title: 'Dashboard'
    },
    children: [
      {
        path: 'analysis',
        name: PageRouterName.DashboardAnalysis,
        component: () => import('@/pages/dashboard/analysis.vue'),
        meta: {
          title: 'Analysis'
        }
      }
    ]
  },
  {
    path: '/system',
    name: PageRouterName.System,
    component: LayoutRoot,
    redirect: '/system/account',
    meta: {
      title: 'System'
    },
    children: [
      {
        path: 'account',
        name: PageRouterName.SystemAccount,
        component: () => import('@/pages/system/account.vue'),
        meta: {
          title: 'SystemAccount'
        }
      }
    ]
  },
  {
    path: '/about',
    name: PageRouterName.About,
    component: LayoutRoot,
    redirect: '/about/index',
    meta: {
      title: 'About'
    },
    children: [
      {
        path: 'index',
        name: PageRouterName.About,
        component: () => import('@/pages/about/index.vue'),
        meta: {
          title: 'About'
        }
      }
    ]
  },
  {
    path: '/:path(.*)*',
    name: PageRouterName.ErrorPage,
    component: LayoutRoot,
    meta: {
      title: 'ErrorPage'
    },
    children: [
      {
        path: '/:path(.*)*',
        name: PageRouterName.ErrorPage,
        component: () => import('@/pages/exception/index.vue'),
        meta: {
          title: 'ErrorPage'
        }
      }
    ]
  }
];

// app router
const router = createRouter({
  // createWebHistory => history路由 
  // createWebHashHistory => hash路由
  history: createWebHashHistory(),
  routes: (routes as unknown) as RouteRecordRaw[],
  strict: true,
  scrollBehavior: () => ({ left: 0, top: 0 }),
})

// config router
export function setupRouter(app: App<Element>) {
  app.use(router);
}

export default router;

值得注意的是，对于404页面的匹配，与之前的只写一个*不同，这里的正则方式匹配有所不同，具体查看官网这里。

路由元数据

在定义的 router/index.ts中，我们给每个路由都添加了一个 meta属性，并给了对应的 title值，用于记录页面的title。这里的 meta 就是路由的元数据，他是一个对象，可以自定义其中的属性。然后在当前路由的 meta 属性上可以访问对应的值。比如我们还可以设置一个 transition 属性，用于设置当前路由的过渡效果；也可以设置一个requiresAuth属性，用于标记当前路由是否可以登录。

导航守卫

正如其名，导航守卫就是通过跳转或者取消的方式来守卫路由。方式有很多，全局前置的、全局后置的、单个路由的、组件级别的。具体的内容参考官方，下面我们按照这个一一实现一个对应的功能。

全局前置守卫

我们需求除了login页面，其他页面都只能是登录了才能访问，而判断登录的条件则是localStorage.getItem('APP_TOKEN')的值不为空，即，已经登录了，并且token存储在本地缓存中。按照这个需求，我们需要在每次路由跳转之前，判断以下两个条件同时满足时，才会跳转正确的页面：

  • 跳转的页面不是login页面
  • 已经登录了

我们创建 @/router/guard/permissionGuard.ts

import type { Router } from 'vue-router';
import { PageRouterName } from '..';

const getAppToken = () => localStorage.getItem('APP_TOKEN');

export function createPermissionGuard(router: Router) {

  router.beforeEach((to, from, next) => {
    
    // 如果设置了页面无需登录，则直接跳转
    if(to.meta.ignoreAuth === true) {
      next();
      return;
    }

    // 登录页跳转到404页面时，重定向到首页
    if(from.name === PageRouterName.Login && to.name === PageRouterName.ErrorPage) {
      next({
        name: PageRouterName.Root
      })
      // 这里的return是必须的，防止执行两次next
      return;
    }

    const isAuthenticated = getAppToken();
    // 判断登录
    if(to.name !== PageRouterName.Login && !isAuthenticated) {
      next({
        name: PageRouterName.Login
      })
      return;
    }

    next();
  })

}

可以看出，这里在路由跳转前的钩子中，进行了一系列的登录判断。只有正确执行 next()，才会正确跳转；

全局后置守卫

按照上面的思路，现在我们需求，当页面跳转到 login 页面时，会清空所有当前的登录信息，即 localStorage.removeItem('APP_TOKEN')

我们创建 @/router/guard/stateGuard.ts

import type { Router } from 'vue-router';
import { PageRouterName } from '..';

const resetState = () => {
  localStorage.removeItem('APP_TOKEN');
}

export function createStateGuard(router: Router) {

  router.afterEach((to) => {
    // 跳转登录页，清空用户信息
    if(to.name === PageRouterName.Login) {
      resetState()
    }
  })
}

单个路由守卫

在设置routes时，除了path、name等参数，还可以传入beforeEnter参数，用于单独处理一个路由。

组件级别守卫

组件内部会提供一些options，用于配置守卫，具体API如下：

  • beforeRouteEnter
  • beforeRouteUpdate
  • beforeRouteLeave
const UserDetails = {
  template: `<div>user</div>`,
  beforeRouteEnter(to, from) {
    // 在渲染该组件的对应路由被验证前调用
    // 不能获取组件实例 `this` ！
    // 因为当守卫执行时，组件实例还没被创建！
  },
  beforeRouteUpdate(to, from) {
    // 在当前路由改变，但是该组件被复用时调用
    // 举例来说，对于一个带有动态参数的路径 `/users/:id`，在 `/users/1` 和 `/users/2` 之间跳转的时候，
    // 由于会渲染同样的 `UserDetails` 组件，因此组件实例会被复用。而这个钩子就会在这个情况下被调用。
    // 因为在这种情况发生的时候，组件已经挂载好了，导航守卫可以访问组件实例 `this`
  },
  beforeRouteLeave(to, from) {
    // 在导航离开渲染该组件的对应路由时调用
    // 与 `beforeRouteUpdate` 一样，它可以访问组件实例 `this`
  },
}

完整应用

比如，我们现在要实现页面路由跳转时，头部会有进度条显示的功能。

进度条功能，我们借助 nprogress，具体文档查看这里

我们再创建 @/router/guard/stateGuard.ts

import type { Router } from 'vue-router';

import nProgress from 'nprogress';

export function createProgressGuard(router: Router) {
  router.beforeEach(async (to) => {
    nProgress.start();
    return true;
  });

  router.afterEach(async () => {
    nProgress.done();
    return true;
  });
}
