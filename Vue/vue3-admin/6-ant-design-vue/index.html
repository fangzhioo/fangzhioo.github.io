前言

之前在开发react项目的时候，接触了一个很棒的UI，就是来自蚂蚁金服团队的 ant design。个人觉得在视觉上，这个款UI要比elementUI的风格更好看 😄，现在他们提供了支持vue3.x的另一个库 ant-design-vue@2。值得注意的是，该库的 2.x 版本才是支持 vue3.x的，1.x需要搭配vue2.x食用。选择这个UI的一部分原因也是为了后续可以做明暗主题。

正文

安装

yarn add ant-design-vue@next

注意安装时，一定要有 @next，才能安装到2.x版本

注册ant-design-vue@2+

我们修改main.ts中的内容

import { createApp } from 'vue';
import App from './App.vue';
import { setupStore } from '@/store';
import { setupRouter } from '@/router';
import { setupRouterGuard } from '@/router/guard';
import Antd from 'ant-design-vue';

import "ant-design-vue/dist/antd.css"; 

const app = createApp(App);

// 注册store
setupStore(app);

// 注册router
setupRouter(app);
// 注册router guard
setupRouterGuard();

// 注册ant-design-vue
app.use(Antd);

app.mount('#app', true);

我们可以在 @/pages/dashboard/analysis.vue的 template中添加以下代码

<template>
  <div>
    <div> username: {{ username || '-' }} </div>
    <div> realName: {{ realName || '-' }} </div>
    <div> roles: {{ roles || '-' }} </div>
    <a-button type="primary">我是按钮</a-button>
    <a-input placeholder="请输入"></a-input>
    <a-rate></a-rate>
  </div>
</template>

ant-design-vue与element-plus组件的默认书写有一点区别的就是，el- 和 a-，其他常用api基本相同。

打包优化

你以为到这里就完事了吗？ 我们可以看到上图中的控制台，有一段警告 You are using a whole package of antd, please use https://www.npmjs.com/package/babel-plugin-import to reduce app bundle size. Not support Vite !!!

这是因为我们使用的是全局导入的方式，官方文档也告诉我们，按需加载需要搭配 babel-plugin-import一起食用。警报也清楚的注明了 Not support Vite !!!，并不支持 vite 中使用。这使得我们打包后的 vendor 文件很大，可以看下面执行 yarn build后的一段结果

可以看到，已经又一个警告了，因为已经超出了 build.chunkSizeWarningLimit 设置的 1500Kb 的标准。那么，我们应该怎么解决这个问题呢？我们知道，vite打包使用的是 rollup，并且配置中，有 build.rollupOptions 选项可以自定义打包配置，那我们需要将 ant-design-vue部分不要打包到vendor中，修改 vite.config.ts中的 build.rollupOptions配置

// ...

export default ({ command, mode }: ConfigEnv): UserConfig => {
  return {
    // ...
    build: {
      // ...
      // chunk 大小警告的限制（以 kbs 为单位）。默认 500
      chunkSizeWarningLimit: 1500,
      rollupOptions: {
        output: {
          entryFileNames: '[name]-[hash].js',
          chunkFileNames: '[name]-[hash].js',
          assetFileNames: '[name]-[hash].[ext]',
          manualChunks(id) {
            if (id.includes('/node_modules/')) {
              // 设置需要独立打包的npm包
              const expansions = ['ant-design-vue'];
              const c = expansions.find(exp => id.includes(`/node_modules/${exp}`));
              if (c) {
                return `expansion-${c}`;
              } else {
                return 'vendor';
              }
            }
          }
        }
      }
    },
    // ...
  };
};

接下来我们再试试运行 yarn build，可以看到，ant-design-vue 被独立打包了出来。包括css也是独立打包的，大大减少了vender的大小。
